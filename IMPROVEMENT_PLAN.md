## 1. 사용자 경험 (UX) 개선

### 1.1 대시보드 개선
**현재 상태**: 기본적인 통계 정보만 표시
**개선 방안**:
- [ ] **실시간 데이터 시각화**
  - Chart.js 또는 D3.js를 활용한 인터랙티브 차트
  - CO2 배출량 추세 그래프 (월별/분기별/연도별)
  - CII 등급 분포 파이 차트
  - Fleet 성과 비교 막대 그래프

- [ ] **위젯 커스터마이징**
  - 사용자가 대시보드 레이아웃을 드래그 앤 드롭으로 조정
  - 위젯 표시/숨김 토글 기능
  - 개인화된 대시보드 설정 저장

- [ ] **Quick Actions 개선**
  - 현재 버튼들이 기능 미구현 → 실제 동작 연결
  - 최근 활동 타임라인 추가
  - 중요 알림 및 경고 표시

### 1.2 Navigation 개선
**현재 상태**: 사이드바 네비게이션만 존재
**개선 방안**:
- [ ] **Breadcrumb 추가**
  - 현재 위치 표시로 사용자 오리엔테이션 향상
  - 빠른 페이지 이동 지원

- [ ] **검색 기능**
  - 전역 검색 바 추가 (선박, 주문, 계산 이력 등)
  - 자동완성 및 필터링 기능
  - 최근 검색 이력 저장

- [ ] **즐겨찾기/북마크**
  - 자주 사용하는 선박이나 계산 설정 즐겨찾기
  - 빠른 접근을 위한 단축키 지원

### 1.3 폼 입력 개선
**현재 상태**: 기본 HTML 폼 요소 사용
**개선 방안**:
- [ ] **실시간 유효성 검사**
  - 입력 중 즉시 피드백 제공
  - 에러 메시지 명확화 및 해결 방법 제시

- [ ] **자동 저장 기능**
  - 계산기 입력 중 임시 저장
  - 세션 복구 기능 (브라우저 종료 후 재개)

- [ ] **입력 도우미**
  - 툴팁 및 도움말 아이콘 추가
  - 예시 데이터 자동 입력 버튼
  - 단위 변환 계산기 (톤 ↔ 킬로그램 등)

### 1.4 피드백 시스템
**현재 상태**: alert() 팝업 사용
**개선 방안**:
- [ ] **토스트 알림 시스템**
  - 비침습적 알림 (화면 우측 상단)
  - 성공/경고/에러 상태별 색상 구분
  - 자동 사라짐 및 수동 닫기 옵션

- [ ] **로딩 상태 표시**
  - API 호출 중 스켈레톤 UI 또는 스피너
  - 진행률 표시 (대용량 데이터 처리 시)

- [ ] **확인 모달 개선**
  - alert() 대신 커스텀 모달 사용
  - 액션 설명 및 되돌릴 수 없는 작업 강조

## 2. 기능 확장 (Feature Enhancement)

### 2.1 Trading 기능 고도화
**현재 상태**: 기본적인 주문 생성/취소/매칭
**개선 방안**:
- [ ] **고급 주문 타입**
  - 지정가 주문 (Limit Order) - 현재 구현됨
  - 시장가 주문 (Market Order) 추가
  - 조건부 주문 (Stop-Loss, Take-Profit)
  - 부분 체결 옵션

- [ ] **거래 이력 및 분석**
  - 개인 거래 내역 상세 페이지
  - 손익 계산 및 통계
  - 거래 성과 차트 (누적 수익률 등)

- [ ] **시장 데이터 개선**
  - 실시간 가격 업데이트 (WebSocket 활용)
  - 호가창 깊이 표시 (Depth Chart)
  - 거래량 및 변동성 지표

- [ ] **알림 시스템**
  - 가격 알림 설정 (목표가 도달 시)
  - 주문 체결 알림 (이메일/푸시)
  - 시장 뉴스 및 규제 업데이트

### 2.2 Calculator 기능 확장
**현재 상태**: CII, EU-ETS, FuelEU 계산 지원
**개선 방안**:
- [ ] **시나리오 분석**
  - 여러 시나리오 동시 비교 (What-if 분석)
  - 최적화 제안 (연료 믹스, 항로 등)
  - 목표 CII 등급 달성을 위한 역산 기능

- [ ] **배치 계산**
  - CSV/Excel 파일 업로드로 다수 선박 일괄 계산
  - 결과 일괄 다운로드 (PDF/Excel)

- [ ] **고급 연료 옵션**
  - 바이오 연료, 암모니아, 수소 등 대체 연료 지원
  - 연료 가격 변동 시뮬레이션
  - 탄소 포집 기술 (CCS) 적용 옵션

- [ ] **규제 준수 체크**
  - 자동 규제 위반 감지
  - 개선 권장사항 제시
  - 규제 변경 사항 자동 반영

### 2.3 Fleet Management 강화
**현재 상태**: 선박 등록/삭제/조회
**개선 방안**:
- [ ] **상세 선박 프로필**
  - 선박 사진 및 문서 첨부
  - 정비 이력 및 검사 일정 관리
  - 승무원 정보 및 인증서 관리

- [ ] **그룹 관리**
  - 선박을 그룹으로 분류 (항로별, 선종별 등)
  - 그룹별 통계 및 비교 분석
  - 태그 시스템 (라벨링)

- [ ] **성능 모니터링**
  - 실시간 AIS 데이터 연동 (가능 시)
  - 연료 효율 추적
  - 이상 징후 감지 및 알림

- [ ] **일정 관리**
  - 항해 일정 캘린더
  - 정비 및 검사 일정 알림
  - 규제 보고 마감일 추적

### 2.4 Reports 모듈 구현
**현재 상태**: 미구현 (플레이스홀더만 존재)
**개선 방안**:
- [ ] **자동 보고서 생성**
  - EU-ETS, DCS, MRV 보고서 템플릿
  - PDF 자동 생성 (jsPDF 활용)
  - 검증자 요구사항 준수

- [ ] **커스텀 보고서**
  - 사용자 정의 보고서 빌더
  - 데이터 필터링 및 정렬
  - 차트 및 표 삽입

- [ ] **보고서 이력**
  - 과거 보고서 아카이브
  - 버전 관리 및 비교
  - 감사 추적 (Audit Trail)

### 2.5 Admin 기능 추가
**현재 상태**: Admin 페이지 미구현
**개선 방안**:
- [ ] **사용자 관리**
  - 사용자 목록 조회 및 검색
  - 역할 변경 (User ↔ Power User ↔ Admin)
  - 계정 활성화/비활성화

- [ ] **시스템 설정**
  - 전역 설정 관리 (기본 연료 가격, 환율 등)
  - 규제 파라미터 업데이트
  - 백업 및 복원 기능

- [ ] **모니터링 대시보드**
  - 시스템 사용 통계 (활성 사용자, 거래량 등)
  - 에러 로그 및 모니터링
  - 성능 메트릭 (API 응답 시간 등)

## 3. 성능 최적화 (Performance Optimization)

### 3.1 프론트엔드 최적화
**현재 상태**: 기본적인 SPA 구조
**개선 방안**:
- [ ] **코드 스플리팅**
  - 라우트별 동적 import
  - 초기 로딩 시간 단축
  - Lazy Loading 적용

- [ ] **이미지 최적화**
  - WebP 포맷 사용
  - 반응형 이미지 (srcset)
  - 이미지 압축 및 CDN 활용

- [ ] **캐싱 전략**
  - Service Worker 활용 (PWA)
  - API 응답 캐싱
  - 정적 자산 브라우저 캐싱

- [ ] **렌더링 최적화**
  - Virtual Scrolling (대량 데이터 리스트)
  - Debounce/Throttle 적용 (검색, 스크롤 이벤트)
  - 불필요한 리렌더링 방지

### 3.2 백엔드 최적화
**현재 상태**: 간단한 Express 서버
**개선 방안**:
- [ ] **데이터베이스 최적화**
  - 인덱싱 전략 수립
  - 쿼리 최적화 및 N+1 문제 해결
  - 연결 풀링 (Connection Pooling)

- [ ] **API 성능 개선**
  - 응답 압축 (gzip)
  - 페이지네이션 구현
  - 필드 선택 (Sparse Fieldsets)

- [ ] **캐싱 레이어**
  - Redis 도입 (세션, 자주 조회되는 데이터)
  - API 응답 캐싱
  - 캐시 무효화 전략

- [ ] **비동기 처리**
  - 무거운 계산 작업 백그라운드 처리
  - 메시지 큐 도입 (Bull, RabbitMQ 등)
  - 이메일 발송 비동기화

### 3.3 모니터링 및 분석
**개선 방안**:
- [ ] **성능 모니터링**
  - Google Analytics 또는 Matomo 통합
  - Core Web Vitals 추적
  - 사용자 행동 분석

- [ ] **에러 추적**
  - Sentry 또는 LogRocket 통합
  - 에러 알림 및 리포팅
  - 소스맵 활용 디버깅

- [ ] **로깅 시스템**
  - 구조화된 로그 (JSON 포맷)
  - 로그 레벨 관리 (debug, info, warn, error)
  - 로그 집계 및 분석 도구

## 4. 보안 강화 (Security Enhancement)

### 4.1 인증 및 권한
**현재 상태**: 기본 localStorage 기반 인증
**개선 방안**:
- [ ] **JWT 토큰 개선**
  - Access Token + Refresh Token 패턴
  - 토큰 만료 시간 설정 및 자동 갱신
  - HttpOnly 쿠키 사용 (XSS 방지)

- [ ] **다중 인증 (MFA)**
  - 이메일 인증 코드
  - TOTP (Google Authenticator 등)
  - SMS 인증 (선택적)

- [ ] **세션 관리**
  - 동시 로그인 제한 또는 관리
  - 활동 없을 시 자동 로그아웃
  - 로그인 이력 추적

- [ ] **역할 기반 접근 제어 (RBAC)**
  - 세밀한 권한 설정 (읽기/쓰기/삭제)
  - 리소스별 접근 제어
  - 권한 상속 및 그룹 관리

### 4.2 데이터 보안
**개선 방안**:
- [ ] **암호화**
  - 전송 중 암호화 (HTTPS 강제)
  - 저장 데이터 암호화 (민감 정보)
  - 비밀번호 해싱 강화 (bcrypt, Argon2)

- [ ] **입력 검증**
  - 서버 사이드 유효성 검사 강화
  - SQL Injection 방지 (Prepared Statements)
  - XSS 방지 (입력 sanitization)

- [ ] **API 보안**
  - Rate Limiting (DDoS 방지)
  - CORS 정책 명확화
  - API 키 관리 (환경 변수)

### 4.3 규정 준수
**개선 방안**:
- [ ] **GDPR 준수**
  - 개인정보 처리 방침 명시
  - 사용자 데이터 다운로드 기능
  - 계정 삭제 및 데이터 완전 삭제

- [ ] **감사 로그**
  - 중요 작업 로그 기록 (삭제, 권한 변경 등)
  - 로그 무결성 보장
  - 정기 감사 리포트

- [ ] **API 통합**
  - 외부 시스템 연동 (AIS 데이터, 연료 가격 API 등)
  - Webhook 지원 (이벤트 알림)
  - RESTful API 문서화 (Swagger/OpenAPI)

  - Service Worker 구현
  - 오프라인 페이지 제공
  - 데이터 동기화 (온라인 복귀 시)

- [ ] **설치 가능**

  - 홈 화면 추가 프롬프트
  - 앱 아이콘 및 스플래시 화면

- [ ] **푸시 알림**

  - 알림 설정 관리
  - 중요 이벤트 알림
